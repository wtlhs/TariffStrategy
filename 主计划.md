## 一、产品定位

**税率政策工具**（Tariff Policy Tool）是一款 Chrome 扩展，面向国际贸易企业的发货策略对比工具。

核心能力：
1. **智能发货策略对比** — 输入 HS编码/发出地/目的地/货值，对比所有可选路线总成本，AI 推荐最优方案
2. **税率数据管理** — 展示 Section 301、MFN 税率、反倾销税等真实数据；管理产品和发货国家
3. **税率变化订阅** — 订阅特定路线/产品的税率变化通知，支持邮件/钉钉/企微/Webhook
4. **积分商业化** — 签到领积分，按需消费，订阅制解锁高级功能

**推荐部署方案（初期低成本）：**
- **API 服务**：Cloudflare Workers（按请求计费，免费额度大）
- **数据库**：Supabase (PostgreSQL) — 含 REST API 自动生成
- **数据采集**：独立 FastAPI 服务 + APScheduler 定时任务
- **支付**：微信支付 + 支付宝（国内）/ Stripe（海外）
- **AI 服务**：Claude API / GPT API 生成策略建议

| 功能 | 离线可用 | 说明 |
|------|---------|------|
| 策略对比（本地模拟） | ✅ | 使用内置模拟数据，完全离线 |
| 策略对比（AI 实时） | ❌ | 需要网络 |
| 产品/发货国家 CRUD | ✅ | 数据存 chrome.storage.local |
| Excel 导入导出 | ✅ | 本地操作 |
| 采集数据展示 | ✅ | 展示缓存数据（标注"上次同步时间"） |
| 订阅规则管理 | ✅ 本地 / ❌ 远程同步 | 本地创建/删除可用，远程同步需网络 |
| 签到 | ❌ | 需要服务端验证 |
| 积分查询 | ⚠️ 展示缓存值 | 标注"离线模式，余额可能不准确" |

离线检测：使用 `navigator.onLine` + `chrome.runtime.onMessage` 监听网络状态变化，顶栏显示离线指示器。

## 五、数据源采集与策略算法

### 5.1 税率数据源

#### 数据源矩阵

| 数据源 | 访问方式 | 覆盖范围 | 免费费率 | 数据类型 | 用途 |
|--------|---------|---------|---------|---------|------|
| **WITS (World Bank)** | REST API | 200+ 国家，HS 6位 | 免费，无需注册 | MFN/优惠税率 | 全局税率基准 |
| **EU TARIC** | REST API | 27 个 EU 成员国 | 免费 | 关税 + 贸易救济 | 欧盟进口税率 |
| **US HTS (USITC)** | 批量下载 (JSON/CSV) | 美国，HTS 10位 | 免费 | 关税税率表 | 美国进口税率 |

### 5.2 策略对比算法

#### 算法演进路线

```
V1 (MVP)     → 简化模型：综合税率 × 货值 + 运费（当前模拟数据）
V2 (上线)    → 增强模型：+ MPF + HMF + 保险 + FTA 优惠
V3 (付费)    → 专业模型：+ AD/CVD + 敏感度分析 + 多维排序
```

#### V1 算法（MVP，纯前端离线）

保持现有模拟数据，仅优化展示：

```
总成本 = 货值 × (1 + 综合税率) + 运费
综合税率 = 关税率 + 惩罚性关税率（预置）
```

模拟数据来源：基于 2026年4月真实税率采集（USITC HTS + USTR Section 301 + FTA 优惠 + Drewry 运费指数）

#### V2 算法（上线后，服务端计算）

**Landed Cost（到岸总成本）完整公式：**

```
到岸总成本 = 货值 + 关税总额 + MPF + HMF + 运费 + 保险

其中：
  关税总额 = MFN关税 + Section301关税 + AD/CVD关税 + Section201关税
  MFN关税   = 货值 × MFN税率
  Section301 = 货值 × Section301税率（仅中国原产）
  AD/CVD    = 货值 × AD/CVD税率（若适用）
  MPF       = clamp(货值 × 0.3464%, $31.67, $614.35)  // 正式进口
  HMF       = 货值 × 0.125%  （仅海运）
  保险      = (货值 + 运费) × 0.5%
```

**FTA 优惠判定逻辑：**

```
if (origin 在 FTA 列表 AND 满足原产地规则):
    适用税率 = min(MFN税率, FTA优惠税率)
    // KORUS → 韩国多数工业品 0%
    // US-Singapore FTA → 新加坡多数 0%
    // USMCA → 墨西哥/加拿大满足规则 0%
```

**De Minimis 判定：**

```
if (货值 ≤ $800 AND 无 AD/CVD AND 无 Section 301):
    关税总额 = 0, MPF = 0, HMF = 0
    // 低值豁免，快递直邮适用
```

**V2 计算实现：**

```typescript
interface LandedCostInput {
  hsCode: string           // HS编码
  origin: string           // 发出地 "CN"
  destination: string      // 目的地 "US"
  goodsValue: number       // 货值 USD
  quantity?: number        // 数量（从量税用）
  weightKg?: number        // 重量 kg（从量税用）
  shippingMode: 'ocean' | 'air' | 'rail' | 'land'
  shippingCost: number     // 运费 USD
}

interface LandedCostResult {
  // 成本拆解
  goodsValue: number
  customsDuty: number       // MFN关税
  section301: number        // Section 301附加
  adCvd: number             // 反倾销/反补贴
  section201: number        // 201保障
  mpf: number               // 商品处理费
  hmf: number               // 港口维护费
  insurance: number         // 保险费
  shippingCost: number      // 运费
  totalCost: number         // 到岸总成本

  // 税率明细
  effectiveRate: number     // 实际综合税率
  mfnRate: number
  section301Rate: number
  adCvdRate: number
  ftaApplied: boolean       // 是否应用了FTA优惠
  ftaName?: string          // "KORUS" 等
  deMinimis: boolean        // 是否低值豁免

  // 数据可信度
  confidence: 'high' | 'medium' | 'low'
  missingData: string[]     // ["AD/CVD 数据缺失"]
}

function calculateLandedCost(
  input: LandedCostInput,
  rates: TariffRate[]
): LandedCostResult {
  // 1. 查找适用税率
  const mfnRate = findRate(rates, input.hsCode, input.origin, 'mfn')
  const s301Rate = findRate(rates, input.hsCode, input.origin, 'section301')
  const adCvdRate = findRate(rates, input.hsCode, input.origin, 'ad_cvd')
  const ftaRate = findRate(rates, input.hsCode, input.origin, 'preferential')

  // 2. De Minimis 判定
  const deMinimis = input.goodsValue <= 800 && !adCvdRate && !s301Rate

  // 3. FTA 优惠判定
  const effectiveMfn = (ftaRate && ftaRate.rate < mfnRate.rate)
    ? ftaRate.rate
    : mfnRate.rate
  const ftaApplied = effectiveMfn < mfnRate.rate

  // 4. 计算各组成部分
  const customsDuty = deMinimis ? 0 : input.goodsValue * effectiveMfn
  const section301 = deMinimis ? 0 : (s301Rate?.rate ?? 0) * input.goodsValue
  const adCvd = (adCvdRate?.rate ?? 0) * input.goodsValue  // AD/CVD 不受 de minimis 豁免
  const mpf = deMinimis ? 0 : Math.max(31.67, Math.min(input.goodsValue * 0.003464, 614.35))
  const hmf = input.shippingMode === 'ocean' ? input.goodsValue * 0.00125 : 0
  const insurance = (input.goodsValue + input.shippingCost) * 0.005

  // 5. 合计
  const totalTax = customsDuty + section301 + adCvd
  const totalCost = input.goodsValue + totalTax + mpf + hmf + input.shippingCost + insurance

  return {
    goodsValue: input.goodsValue,
    customsDuty, section301, adCvd, section201: 0,
    mpf, hmf, insurance,
    shippingCost: input.shippingCost,
    totalCost,
    effectiveRate: totalTax / input.goodsValue,
    mfnRate: mfnRate.rate,
    section301Rate: s301Rate?.rate ?? 0,
    adCvdRate: adCvdRate?.rate ?? 0,
    ftaApplied,
    ftaName: ftaApplied ? ftaRate?.notes : undefined,
    deMinimis,
    confidence: determineConfidence(rates, input),
    missingData: findMissingData(rates, input),
  }
}
```

#### V3 算法（付费，专业模式）

**多维度评分排序：**

```typescript
interface RouteScore {
  route: string
  landedCost: number
  landedCostBest: number    // 最优情况（FTA 等）
  landedCostWorst: number   // 最差情况（含 AD/CVD 估算）
  transitDays: number
  riskScore: number         // 0-100（税率波动风险）
  complianceScore: number   // 0-100（合规复杂度）
  costSensitivity: number   // 1%税率变化对总成本的影响
}

function rankRoutes(routes: RouteScore[]): RouteScore[] {
  return routes.sort((a, b) => {
    // 成本差 > 5%：成本优先
    if (Math.abs(a.landedCost - b.landedCost) > a.landedCost * 0.05) {
      return a.landedCost - b.landedCost
    }
    // 成本差 ≤ 5%：综合评分
    const scoreA = a.landedCost * 0.6 + a.riskScore * 10 + a.transitDays * 50
    const scoreB = b.landedCost * 0.6 + b.riskScore * 10 + b.transitDays * 50
    return scoreA - scoreB
  })
}
```

**敏感度分析：**

```
对于 $50,000 货值：
· 1% 税率变化 → $500 成本变化
· Section 301 10% 变化 → $5,000 成本变化
· AD/CVD 新增 50% → $25,000 成本变化

用途：
· "如果美国对中国加征 10% 关税，RO→US 仍是最优吗？"
· "AD/CVD 立案调查后，这条路线还有优势吗？"
· "运费上涨 20%，最优路线会变化吗？"
```

function calculateRiskScore(route: RouteData): number {
  let risk = 0
  // 有 Section 301 历史 → 高风险
  if (route.section301Rate > 0) risk += 30
  // 有 AD/CVD 调查 → 高风险
  if (route.underAdCvdReview) risk += 25
  // FTA 依赖 → 中等风险（规则可能变化）
  if (route.ftaApplied) risk += 15
  // 地缘政治风险
  risk += getGeopoliticalRisk(route.origin)
  return Math.min(risk, 100)
}

- 免费额度耗完：展示用量报告"本周您已节省 ¥X,XXX"
- 功能灰化点击：弹出套餐对比表 + 7天试用入口
- 签到后：底部文案"专业版签到 2x 加成"
- 试用期到期前 3 天：推送通知
- **不自动扣费**，试用到期自动降级

## 十、隐私政策要点（PRIVACY.md）

Chrome Web Store 上架强制要求，核心内容：

1. **数据收集范围：** 邮箱（注册用）、产品配置数据、发货国家配置、搜索历史
2. **不收集：** 浏览历史、Cookie、密码、支付信息（支付由第三方处理）
3. **数据存储：** 本地 chrome.storage.local（默认）、服务端数据库（注册用户）
4. **第三方服务：** AI 税率服务（用户自行配置地址）、支付平台（微信/支付宝/Stripe）
5. **数据删除：** 用户可随时导出并删除所有数据
6. **儿童隐私：** 不面向 13 岁以下用户
7. **联系方式：** 开发者邮箱

## 十二、风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| WXT 框架学习成本 | 延迟 1-2 天 | all-api-hub 代码可参考 |
| Chrome popup 600px 高度限制 | 布局受限 | 折叠策略 + "展开详情"跳全屏 |
| AI 服务不可用 | 高级功能无法使用 | 本地模拟数据降级 |
| 积分系统并发安全 | 超额消耗 | 服务端幂等校验 + 乐观锁 |
| Chrome Web Store 审核周期 | 上线延迟 1-3 天 | 提前准备素材，先发测试版 |
| MV3 Service Worker 5分钟超时 | 长任务中断 | 使用 chrome.alarms 而非 setTimeout |
| 付费转化率低于预期 | 收入不达预期 | 加强签到日活 + 优化转化触点 |
