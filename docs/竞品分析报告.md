# 竞品分析与迭代计划：税率政策工具

> 调研日期：2026-04-27  
> 适用范围：当前项目 `TariffStrategy`，重点面向中国出海企业、美国进口关税测算、原产地路径对比、税率变化监控。  
> 注意：本报告用于产品和研发决策，不构成法律、税务或报关建议。

## 一、核心结论

当前项目的机会不在于做一个“大而全”的跨境税务或物流平台，而在于做一个更聚焦的工具：

**面向中国出海企业的美国进口关税决策与监控工具。**

最值得保留的差异化方向是：

- 中文原生，服务采购、外贸、供应链和财务人员，而不是只服务欧美报关团队。
- Chrome 扩展 + Excel 批量工作流，贴近日常选品、询价、采购和报价场景。
- 对美国多层关税进行可解释拆解：MFN/FTA、Section 301、Section 232、AD/CVD、Section 122、MPF/HMF。
- 多原产地并排对比，用于回答“从中国、越南、韩国、墨西哥、加拿大等地发货/生产，成本差多少”。
- 税率变化订阅，用于持续跟踪某些 HS 编码和路线的政策风险。

原报告中“唯一提供 5 层关税叠加”“唯一支持多原产地路径对比”的表述需要降级。Flexport、DutyDesk、TariffsAPI 等已经提供多层税率拆解、来源国对比、API、告警或 Chrome/Sheets 集成。我们的可防守定位应改为：

**中文 + 浏览器工作流 + Excel 批量 + 美国政策变动解释 + 面向中国企业的原产地策略工具。**

## 二、2026 年政策环境变化

这部分会直接影响计算引擎和产品文案，优先级高于新增 UI 功能。以下内容为截至调研日的公开信息摘要，实际适用以 CBP、USITC、HTSUS、Federal Register 和白宫后续发布为准。

| 政策项 | 截至 2026-04-27 的状态 | 对产品的影响 |
|--------|------------------------|--------------|
| IEEPA / Reciprocal Tariffs | 2026-02-20，美国最高法院在 `Learning Resources, Inc. v. Trump` 中认定 IEEPA 不授权总统征收关税。IEEPA 关税不应作为当前生效税种继续默认计入。 | 当前代码中的 `reciprocalTariff` 应改为“历史/退款/失效税种”，保留历史查询，不默认计入当前成本。 |
| Section 122 临时进口附加税 | 白宫 2026-02-20 公告使用 Trade Act of 1974 Section 122 征收临时进口附加税；多家贸易合规机构跟踪为 2026-02-24 生效、150 天期限、通常 10% 且有豁免。 | 需要新增 `section122` 税种，并支持生效日、失效日、豁免清单、与 Section 232、Chapter 99 及其他附加税的叠加/排除规则。 |
| De minimis | CBP 于 2025-08-29 开始全球执行 de minimis 暂停；中国/香港曾于 2025-05-02 先行暂停。 | 不能只写“CN/HK 已撤销”。默认应为美国 de minimis 全球暂停，保留历史区间和例外说明。 |
| Section 232 | Flexport 与 CBP 页面显示 Section 232 覆盖汽车及零部件、钢铝及衍生品等，且存在材料含量、Chapter 99、国家/协议例外。 | 当前 HS4 前缀映射太粗。应支持 HTS10、材料占比、Chapter 99、Section 232 与 Section 122 及其他附加税的叠加/排除规则。 |
| Section 301 | 对中国原产仍是核心税种，且有清单、排除、恢复、到期等状态。 | 需要从 USTR/CBP/Federal Register 获取清单、排除码、有效期，不应只靠静态 HS4 映射。 |
| AD/CVD | 依赖产品、原产国、生产商/出口商、现金保证金率和复审结果。 | 不适合作为简单 HS 前缀税率。P0 阶段不应自动计入总成本，应先作为“可能适用，需人工确认”的风险提示；P1 再做订单/企业维度数据。 |

## 五、当前项目现状判断

从本仓库现状看，项目已经不是纯概念阶段：

- 前端为 WXT + React + TypeScript Chrome 扩展形态。
- `src/lib/strategy-engine.ts` 已实现本地 landed cost 计算，包括 MFN/FTA、Section 301、Section 232、AD/CVD 占位、reciprocal tariff、MPF、HMF、保险、运费。
- `src/lib/mock-data.ts` 已有 5 个演示 HS 品类、9 个原产地、FTA、Section 301/232 样本和数据来源说明。
- 后端目录已有 Supabase、支付、通知和采集器雏形，包括 `ushts`、`ustr`、`wits`、`federal-register` 等采集服务。
- 通知服务已经考虑邮件、钉钉、企业微信和 webhook。
- 商业化已有积分、套餐、签到、支付架构。

但如果目标是做可信的贸易决策工具，当前最大短板不是 UI，而是数据模型和政策正确性：

1. `reciprocalTariff` 命名已经过时。IEEPA 应作为历史税种，当前默认应是 Section 122 临时附加税。
2. `DE_MINIMIS_BY_COUNTRY.default` 仍保留 `$800 enabled: true` 的思路，与 2025-08-29 后美国全球暂停不一致。
3. Section 232 不能用 HS4 粗略套用，至少需要 HTS10 + 材料含量 + Chapter 99 + 豁免规则。
4. AD/CVD 不能只按 HS 编码判断，需要原产国、生产商/出口商、订单、现金保证金率和复审状态。
5. 报告和 UI 需要把“税率来源、抓取时间、生效日期、失效日期、置信度、缺失项”作为核心展示项。

## 六、推荐产品定位

### 一句话定位

**给中国出海企业用的美国进口关税测算、原产地对比和政策变动监控工具。**

### 不建议短期承诺

- 不承诺替代报关行或律师。
- 不承诺 DDP 税费保证。
- 不承诺税务代缴。
- 不承诺 AI HS 分类一定准确或可作为 binding ruling。

### 应该重点承诺

- 每次计算展示完整拆解和来源。
- 支持多原产地、批量货值、不同入境日期的对比。
- 明确标注数据更新时间、适用区间和不确定项。
- 中文解释美国政策变化对成本的影响。
- 支持导出可给老板、客户、采购或财务看的报告。

## 七、产品改进优先级

### P0-A：政策正确性热修

目标：先避免当前产品输出明显过时或误导性的结果。

| 改进项 | 具体动作 | 验收标准 |
|--------|----------|----------|
| IEEPA 历史化 | `reciprocalTariff` 不再作为当前默认生效税种计入成本，改为 IEEPA 历史/退款/失效税种。 | 当前日期计算不再默认计入 IEEPA；历史 `as_of` 可保留查询入口。 |
| Section 122 最小支持 | 新增 `section122`，支持自 2026-02-24 起按 150 天临时期限计算、10% 临时附加税、豁免清单、与 Section 232、Chapter 99 及其他附加税的叠加/排除关系；实际终止日以官方后续公告为准。 | 输入不同入境日期，结果能正确切换 IEEPA 历史、Section 122 当前和过期状态。 |
| De minimis 修正 | 默认美国 de minimis 全球暂停；保留 2025-05-02 CN/HK 和 2025-08-29 全球两个历史节点。 | 低于 $800 的美国进口不再默认免税，UI 显示政策原因。 |
| AD/CVD 风险提示 | P0 不自动计入 AD/CVD 总成本，只展示“可能适用，需人工确认”的风险提示。 | 结果页不把不确定 AD/CVD 当作确定税额，同时显示人工确认说明。 |
| 文案同步 | 将“对等关税”“CN/HK de minimis 已撤销”“AI 推荐最优方案”等口径改成更谨慎表述。 | UI、i18n 和报告文案不再出现过时或过度承诺口径。 |

### P0-B：可信计算模型

目标：建立长期可维护、可追溯的关税计算底座。

| 改进项 | 具体动作 | 验收标准 |
|--------|----------|----------|
| 税种模型重构 | 新增统一 `TariffMeasure`：`type`、`legalBasis`、`chapter99Code`、`rate`、`effectiveDate`、`expiryDate`、`stackingRule`、`exemptions`、`sourceUrl`、`confidence`。 | 所有税种均可解释来源和有效期。 |
| 来源面板 | 每个计算结果展示 USITC/USTR/CBP/Federal Register 等来源、抓取时间和缺失项。 | 用户能看到“为什么是这个税率”。 |
| 回归测试样本 | 用 10-20 个固定 HTS10 + 原产地 + 日期样本建立回归测试。 | `pnpm test` 覆盖核心叠加规则。 |
| 缺失项机制 | 当材料占比、生产商、AD/CVD scope、Chapter 99 等信息不足时，结果显式标注缺失字段。 | 系统不会用静默默认值伪造确定性。 |

### P1：数据采集与变化监控

目标：从演示数据升级为可运营的数据服务。

数据源优先级建议：

1. **USITC HTS / HTSUS**：基础税率、special rate、Chapter 99 结构。
2. **USTR**：Section 301 清单、排除、恢复和到期。
3. **CBP**：执行指引、de minimis、Entry Summary / Chapter 99 使用说明。
4. **Federal Register**：AD/CVD、Section 232、临时措施公告。
5. **ITA ADCVD**：AD/CVD 案件、现金保证金率、生产商/出口商维度。

| 改进项 | 具体动作 | 验收标准 |
|--------|----------|----------|
| USITC HTS 数据导入 | 建立 HTS10 基础税率、special rate、Chapter 99 关联导入。 | 覆盖常用章节，不再只靠 5 个 mock 品类。 |
| USTR/CBP/Federal Register 变化流 | 采集 Section 301、Section 232、AD/CVD、Section 122 相关公告，并入 `tariff_change_log`。 | 数据中心可展示政策变更时间线。 |
| 数据健康看板 | 展示每个来源最近同步时间、失败次数、记录数、异常项。 | 用户和运营都能识别数据是否过期。 |
| 订阅规则落地 | 支持按 HS、原产地、税种、成本变化阈值订阅。 | 某个税率变动后可通过邮件/钉钉/企微/webhook 推送。 |

### P2：工作流增强

目标：把工具从“查一次”变成“日常决策流程”。

| 改进项 | 具体动作 | 验收标准 |
|--------|----------|----------|
| 批量对比报告 | Excel 导入多个 SKU、多个原产地、多个货值，导出 5 Sheet 报告。 | 采购可直接上传报价表并得到路线排名。 |
| What-if 模拟 | 支持税率 +5%、运费 +20%、Section 232 新增、FTA 失效等情景。 | 用户能判断最优路线是否稳定。 |
| 历史 `as_of` 查询 | 根据入境日期回放历史税率。 | 支持 IEEPA 退款、旧订单复盘和审计。 |
| 报告导出 | 导出中文 PDF/Excel，含假设、来源、风险和建议。 | 可用于内部审批或客户报价沟通。 |

### P3：AI 分类与 API 商业化

目标：补齐竞品能力，但必须建立在可信数据之上。

| 改进项 | 具体动作 | 验收标准 |
|--------|----------|----------|
| AI HS/HTS 分类 | 基于产品描述、材质、用途、图片/发票字段，返回 3 个候选 HTS10、置信度、GRI/章节注释/CBP ruling 依据。 | AI 不直接“一锤定音”，能提出澄清问题。 |
| REST API | 提供 `/resolve`、`/compare`、`/batch`、`/watch`、`/changes`。 | 返回每层税种、来源、有效期、警告和置信度。 |
| Webhook | 付费用户可接收税率变化通知。 | 与当前通知服务打通。 |
| Shopify/电商插件评估 | 只做 duty preview，不承诺 DDP/代缴。 | 作为 P3/P4，不影响核心工具主线。 |

## 八、12 周迭代路线图

12 周目标应收敛为：可信计算底座 + 官方数据管道 + 批量报告 + 订阅闭环。AI 分类和 API 商业化应作为后续阶段，取决于 P0/P1 的数据可信度完成情况。

| 时间 | 目标 | 主要交付 |
|------|------|----------|
| 第 1-2 周 | 政策正确性热修 | IEEPA 历史化、Section 122 最小支持、de minimis 全球暂停、AD/CVD 风险提示、文案同步。 |
| 第 3-4 周 | 可信计算模型 | `TariffMeasure`、来源字段、有效期、置信度、缺失项机制、核心回归测试。 |
| 第 5-6 周 | 官方数据管道 | USITC HTS 导入、USTR/CBP/Federal Register 增量、数据健康看板。 |
| 第 7-8 周 | 批量对比报告 | Excel 批量测算、多原产地路线排名、风险提示、中文导出报告。 |
| 第 9-10 周 | 变化订阅闭环 | HS/路线 watchlist、通知渠道、政策变化时间线、成本变化阈值提醒。 |
| 第 11-12 周 | 试点与校验 | 内部样本校验、用户试点、结果解释优化、付费套餐灰度。 |

后续候选阶段：AI HS/HTS 分类、REST API、Webhook、Shopify/电商插件评估。只有在官方数据源、来源追溯和回归样本稳定后，再进入商业化扩展。

## 九、商业化建议

原计划中的 Free/Starter/Pro/Enterprise 方向合理，但价格需要结合数据成本和竞品重新校准。

| 档位 | 建议 | 理由 |
|------|------|------|
| Free | 保留每日/每周少量计算，允许查看拆解但限制批量、历史和订阅。 | 对标 SimplyDuty 5 次/天、DutyDesk 5 次/月，降低试用门槛。 |
| Starter | ¥49/月左右，面向个人采购/外贸。 | 支持本地无限对比、少量 AI、少量订阅、Excel 导出。 |
| Pro | ¥199/月左右，面向中小贸易公司。 | 竞品 TariffsAPI Basic 为 $49/月，DutyDesk Business 为 $59/月；如果提供可信数据和中文报告，¥199 有空间。 |
| Enterprise | 定制，按团队席位/API/私有部署计费。 | 企业用户关注批量、API、审计、SLA 和专属支持。 |

短期不建议以“积分签到”作为核心商业化叙事。贸易工具用户更关心准确、及时、可追溯。积分可以作为促活机制，但主价值应该是：

- 减少报关/咨询前的预判成本。
- 避免报价低估关税。
- 及时发现政策变化。
- 快速比较供应链转移方案。

## 十、需要立即修改的文案口径

| 原口径 | 建议口径 |
|--------|----------|
| 唯一提供 5 层美国关税叠加计算 | 中文场景下提供可解释的美国多层关税拆解和原产地对比。 |
| 唯一支持多原产地路径对比 | 面向采购和供应链决策，优先支持多原产地并排测算。 |
| 对等关税 | 改为“IEEPA 历史关税 / Section 122 临时附加税”，按生效日期区分。 |
| CN/HK de minimis 已撤销 | 美国 de minimis 已于 2025-08-29 全球暂停，CN/HK 于 2025-05-02 先行暂停。 |
| AI 推荐最优方案 | 改为“基于当前数据和假设生成建议，并展示缺失数据和风险”。 |

## 十一、关键风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 政策快速变化 | 计算结果过期，用户误用。 | 数据健康看板、结果页强制显示更新时间和适用日期。 |
| AD/CVD 复杂度过高 | 误算风险高。 | P0 不自动计入总成本，只提示“可能适用，需人工确认”；P1 再接入 ITA/Federal Register/人工校验。 |
| AI 分类误判 | 合规和成本风险。 | 返回候选、依据、置信度和澄清问题，不做 binding claim。 |
| 竞品功能成熟 | “Chrome 扩展/多层叠加”不再独特。 | 坚守中文、Excel、采购决策和中国企业工作流。 |
| 数据授权和爬取稳定性 | 服务不可用或数据缺失。 | 优先使用官方开放数据/API，记录来源和同步状态。 |

## 十二、推荐下一步

优先做 3 件事：

1. **修正计算引擎政策模型**：Section 122、IEEPA 历史状态、de minimis 全球暂停、来源和有效期字段。
2. **补数据可信度闭环**：官方来源导入、更新时间、缺失项、回归测试样本。
3. **把批量对比报告做成主卖点**：Excel 上传、多原产地结果、中文解释、导出报告。

这三件事完成后，当前项目才适合对外宣称“美国关税决策工具”。在此之前，更准确的说法是“具备核心计算框架的 Chrome 端测算原型，仍需补齐官方数据源和政策追溯能力”。
