# 支付系统架构设计

> 税率政策工具 Chrome 扩展 — 支付功能技术方案
>
> 最后更新：2026-04-24

---

## 一、方案概述

### 核心思路

Chrome Extension 由于 Manifest V3 的 CSP 限制，无法内嵌第三方支付 SDK。采用 **外部支付页跳转** 方案：

```
扩展内点击"购买" → 新 Tab 打开支付网页 → 用户扫码/付款 → 扩展感知支付结果
```

### 商户资质要求

| 渠道 | 要求 | 申请周期 |
|------|------|---------|
| 微信支付 | 营业执照 + 微信支付商户号 | 1-5 个工作日 |
| 支付宝 | 营业执照 + 支付宝商户号 | 1-3 个工作日 |
| Stripe（预留） | 个人即可，邮箱注册 | 即时 |

> **注意**：微信支付和支付宝均要求企业资质（个体工商户亦可）。如暂无资质，可先用 Stripe 跑通流程，后续补接国内支付。

## 三、数据库设计

### 3.1 用户表 (users)

```sql
CREATE TABLE users (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  phone           VARCHAR(20) UNIQUE,            -- 手机号（主登录方式）
  email           VARCHAR(255),                  -- 邮箱（可选绑定）
  plan            VARCHAR(20) DEFAULT 'free',    -- free/starter/pro/enterprise
  plan_expires_at TIMESTAMPTZ,                   -- 套餐到期时间
  trial_ends_at   TIMESTAMPTZ,                   -- 试用到期时间
  credits         INTEGER DEFAULT 100,           -- 积分余额
  total_earned    INTEGER DEFAULT 100,           -- 累计获得积分
  total_spent     INTEGER DEFAULT 0,             -- 累计消耗积分
  checkin_streak  INTEGER DEFAULT 0,             -- 连续签到天数
  last_checkin_at TIMESTAMPTZ,                   -- 上次签到时间
  is_online       BOOLEAN DEFAULT true,
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  updated_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_users_phone ON users(phone);
```

### 3.2 支付订单表 (payment_orders)

```sql
CREATE TABLE payment_orders (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID NOT NULL REFERENCES users(id),
  order_no        VARCHAR(32) UNIQUE NOT NULL,   -- 业务订单号 TFP20260424xxxxx
  order_type      VARCHAR(20) NOT NULL,          -- credits / plan
  status          VARCHAR(20) DEFAULT 'pending', -- pending / paid / failed / refunded / expired

  -- 订单内容
  amount_cents    INTEGER NOT NULL,              -- 支付金额（分）¥39.00 → 3900
  currency        VARCHAR(3) DEFAULT 'CNY',

  -- 积分购买
  credit_amount   INTEGER,                       -- 购买积分数（order_type=credits 时）

  -- 套餐购买
  plan_tier       VARCHAR(20),                   -- starter / pro / enterprise
  plan_cycle      VARCHAR(10),                   -- monthly / yearly
  plan_months     INTEGER,                       -- 购买月数

  -- 支付渠道
  payment_channel VARCHAR(20),                   -- wechat / alipay / stripe
  payment_no      VARCHAR(64),                   -- 三方支付流水号

  -- 时间
  paid_at         TIMESTAMPTZ,
  expired_at      TIMESTAMPTZ,                   -- 订单过期时间（30分钟）
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  updated_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_orders_user ON payment_orders(user_id);
CREATE INDEX idx_orders_status ON payment_orders(status);
CREATE INDEX idx_orders_no ON payment_orders(order_no);
```

### 3.3 积分流水表 (credit_transactions)

```sql
CREATE TABLE credit_transactions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID NOT NULL REFERENCES users(id),
  order_id        UUID REFERENCES payment_orders(id), -- 关联订单（可为空，签到等无订单）
  type            VARCHAR(10) NOT NULL,          -- earn / spend
  amount          INTEGER NOT NULL,              -- 正数
  reason          VARCHAR(50) NOT NULL,          -- purchase / checkin / consume_compare / ...
  balance_after   INTEGER NOT NULL,              -- 操作后余额
  created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_credits_user ON credit_transactions(user_id);
CREATE INDEX idx_credits_created ON credit_transactions(created_at);
```

### 3.4 套餐订阅表 (plan_subscriptions)

```sql
CREATE TABLE plan_subscriptions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID NOT NULL REFERENCES users(id),
  order_id        UUID REFERENCES payment_orders(id),
  plan_tier       VARCHAR(20) NOT NULL,
  cycle           VARCHAR(10) NOT NULL,          -- monthly / yearly
  status          VARCHAR(20) DEFAULT 'active',  -- active / cancelled / expired
  started_at      TIMESTAMPTZ DEFAULT NOW(),
  expires_at      TIMESTAMPTZ NOT NULL,
  auto_renew      BOOLEAN DEFAULT false,
  cancelled_at    TIMESTAMPTZ,
  created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_plan_sub_user ON plan_subscriptions(user_id);
```

## 四、支付渠道接入

### 4.1 微信支付

**接入方式**: Native 支付（扫码支付）

```
用户下单 → 后端调用微信统一下单API → 返回二维码URL → 前端展示二维码
用户扫码付款 → 微信服务器回调后端 → 后端确认支付 → 发放积分/升级套餐
```

**后端关键参数**:

```typescript
// 微信支付配置
const WECHAT_PAY_CONFIG = {
  appId: process.env.WECHAT_APP_ID,           // 公众号/小程序 AppID
  mchId: process.env.WECHAT_MCH_ID,           // 商户号
  apiKey: process.env.WECHAT_API_KEY,         // API 密钥 (V3)
  serialNo: process.env.WECHAT_SERIAL_NO,     // 证书序列号
  privateKey: fs.readFileSync('./certs/wechat/apiclient_key.pem'),
  notifyUrl: 'https://api.tarifftool.com/api/payments/wechat/notify',
}
```

**统一下单接口**: `POST https://api.mch.weixin.qq.com/v3/pay/transactions/native`

```typescript
// 创建预支付订单
async function createWechatOrder(order: PaymentOrder) {
  const body = {
    appid: WECHAT_PAY_CONFIG.appId,
    mchid: WECHAT_PAY_CONFIG.mchId,
    description: order.order_type === 'credits'
      ? `税率工具 ${order.credit_amount} 积分`
      : `税率工具 ${order.plan_tier} 套餐`,
    out_trade_no: order.order_no,
    notify_url: WECHAT_PAY_CONFIG.notifyUrl,
    amount: {
      total: order.amount_cents,    // 分
      currency: 'CNY',
    },
    time_expire: order.expired_at?.toISOString(),
  }

  // 签名 + 发送请求
  const response = await signedRequest('POST', '/v3/pay/transactions/native', body)
  return response.code_url  // 二维码链接
}
```

**回调处理**:

```typescript
// 微信支付回调
async function handleWechatNotify(body: WechatNotifyBody) {
  // 1. 验证签名（防止伪造）
  verifyWechatSignature(body)

  // 2. 解密通知内容
  const decryptResult = decryptWechatResource(body.resource)

  // 3. 幂等检查：订单是否已处理
  const order = await findOrderByNo(decryptResult.out_trade_no)
  if (order.status === 'paid') return { code: 'SUCCESS', message: 'OK' }

  // 4. 确认支付成功
  if (decryptResult.trade_state === 'SUCCESS') {
    await fulfillOrder(order, {
      paymentNo: decryptResult.transaction_id,
      paidAt: new Date(),
    })
  }

  return { code: 'SUCCESS', message: 'OK' }
}
```

**接入方式**: 当面付（扫码支付）


**统一下单**: 使用 `alipay.trade.precreate` 接口

```typescript
import AlipaySdk from 'alipay-sdk'

const alipaySdk = new AlipaySdk(ALIPAY_CONFIG)

async function createAlipayOrder(order: PaymentOrder) {
  const result = await alipaySdk.exec('alipay.trade.precreate', {
    notifyUrl: ALIPAY_CONFIG.notifyUrl,
    bizContent: {
      out_trade_no: order.order_no,
      total_amount: (order.amount_cents / 100).toFixed(2),  // 元
      subject: order.order_type === 'credits'
        ? `税率工具 ${order.credit_amount} 积分`
        : `税率工具 ${order.plan_tier} 套餐`,
      timeout_express: '30m',
    },
  })
  return result.qrCode  // 二维码内容
}
```

**回调处理**:

```typescript
async function handleAlipayNotify(params: AlipayNotifyParams) {
  // 1. 验证签名
  if (!alipaySdk.checkNotifySign(params)) {
    throw new Error('支付宝签名验证失败')
  }

  // 2. 幂等检查
  const order = await findOrderByNo(params.out_trade_no)
  if (order.status === 'paid') return 'success'

  // 3. 确认支付成功
  if (params.trade_status === 'TRADE_SUCCESS') {
    await fulfillOrder(order, {
      paymentNo: params.trade_no,
      paidAt: new Date(),
    })
  }

  return 'success'
}
```

### 5.2 订单履行（支付成功后）

```typescript
/**
 * 支付成功后的统一处理逻辑
 * 必须保证幂等性（重复调用不会重复发放）
 */
async function fulfillOrder(order: PaymentOrder, paymentInfo: PaymentInfo) {
  // 使用数据库事务保证原子性
  await prisma.$transaction(async (tx) => {
    // 1. 更新订单状态
    const updated = await tx.paymentOrder.update({
      where: { id: order.id },
      data: {
        status: 'paid',
        paymentChannel: paymentInfo.channel,
        paymentNo: paymentInfo.paymentNo,
        paidAt: paymentInfo.paidAt,
      },
    })

    // 幂等：如果已经被处理过（status 已变），直接返回
    if (updated.status === 'paid' && updated.paidAt !== paymentInfo.paidAt) {
      return
    }

    // 2. 根据订单类型发放
    if (order.orderType === 'credits') {
      // 发放积分
      const user = await tx.user.update({
        where: { id: order.userId },
        data: {
          credits: { increment: order.creditAmount },
          totalEarned: { increment: order.creditAmount },
        },
      })

      // 记录流水
      await tx.creditTransaction.create({
        data: {
          userId: order.userId,
          orderId: order.id,
          type: 'earn',
          amount: order.creditAmount,
          reason: 'purchase',
          balanceAfter: user.credits,
        },
      })
    }

    if (order.orderType === 'plan') {
      // 升级套餐
      const expiresAt = calculatePlanExpiry(order.planCycle)

      // 取消当前有效订阅
      await tx.planSubscription.updateMany({
        where: { userId: order.userId, status: 'active' },
        data: { status: 'cancelled', cancelledAt: new Date() },
      })

      // 创建新订阅
      await tx.planSubscription.create({
        data: {
          userId: order.userId,
          orderId: order.id,
          planTier: order.planTier,
          cycle: order.planCycle,
          expiresAt,
        },
      })

      // 更新用户套餐
      await tx.user.update({
        where: { id: order.userId },
        data: { plan: order.planTier, planExpiresAt: expiresAt },
      })
    }
  })
}
```

```typescript
// 扩展端：支付状态轮询
async function pollOrderStatus(orderId: string): Promise<'paid' | 'expired'> {
  const MAX_ATTEMPTS = 60        // 最多 60 次
  const INTERVAL_MS = 3000       // 每 3 秒

  for (let i = 0; i < MAX_ATTEMPTS; i++) {
    await sleep(INTERVAL_MS)

    const result = await apiClient.get(`/orders/${orderId}/status`)

    if (result.status === 'paid') {
      // 刷新本地状态
      await refreshUserState()
      showToast('充值成功！')
      return 'paid'
    }

    if (result.status === 'expired') {
      showToast('订单已过期')
      return 'expired'
    }
  }

  showToast('支付超时，请重试')
  return 'expired'
}
```

## 八、安全设计

### 8.1 支付安全

| 风险 | 防护措施 |
|------|---------|
| 订单伪造 | 后端签名 + orderId 不透明（UUID） |
| 重复支付回调 | 幂等处理，检查订单状态后再发放 |
| 金额篡改 | 回调中校验金额与订单一致 |
| 订单盗用 | orderId 绑定 userId，回调中验证归属 |
| 暴力轮询 | 限流：同一用户 1次/3秒 |
| 订单过期 | 30分钟自动关闭，定时任务清理 |

## 九、扩展端对接改动

### 9.1 购买积分按钮

```typescript
// 在 PlanPage 或积分浮层中
async function handleBuyCredits(pack: CreditPack) {
  const userId = userStore.getState().user?.userId
  if (!userId) {
    // 未登录，引导登录
    showPhoneLoginModal()
    return
  }

  try {
    const result = await apiClient.post('/orders/create', {
      userId,
      orderType: 'credits',
      creditPackId: pack.id,
      paymentChannel: 'wechat',  // 默认微信，可在支付页切换
    })

    // 打开支付页
    window.open(result.paymentPageUrl, '_blank')

    // 开始轮询
    pollOrderStatus(result.orderId)
  } catch (err) {
    showToast('创建订单失败，请重试')
  }
}
```
