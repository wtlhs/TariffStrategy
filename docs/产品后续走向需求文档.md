# 产品后续走向需求文档：美国关税决策与监控工具

> 文档日期：2026-04-28  
> 适用项目：`TariffStrategy`  
> 关联文档：`docs/竞品分析报告.md`  
> 文档性质：产品方向 PRD，用于后续研发排期、功能取舍和商业化验证。  
> 免责声明：本产品用于贸易成本测算、政策监控和内部决策辅助，不构成法律、税务、报关或海关归类意见。

## 一、产品定位

### 1. 一句话定位

给中国出海企业用的美国进口关税测算、原产地对比和政策变动监控工具。

### 2. 核心价值

帮助采购、外贸、供应链、财务和经营者在报价、选品、供应链转移和订单复盘时，快速回答：

- 这个 HS/HTS 商品当前进口美国需要承担哪些关税和费用？
- 从中国、越南、韩国、墨西哥、加拿大等不同原产地进口，成本差多少？
- 某个政策变化会让哪些 SKU、路线或客户报价受影响？
- 当前测算结果是否可靠，来源是什么，哪些字段还需要人工确认？
- 能否导出一份老板、客户、采购或财务能看懂的中文报告？

### 3. 不做什么

短中期不做以下承诺：

- 不替代报关行、律师、税务顾问或 binding ruling。
- 不承诺 DDP 税费保证。
- 不做税务代缴。
- 不把不确定的 AD/CVD 自动计入确定成本。
- 不把 AI 分类结果作为最终 HTS 归类结论。
- 不优先做全球全品类税务合规平台。

## 二、目标用户

| 用户角色 | 典型场景 | 核心痛点 | 产品价值 |
|----------|----------|----------|----------|
| 外贸业务 | 给美国客户报价、评估利润 | 不知道政策变化后关税是否吃掉利润 | 快速测算 landed cost，导出报价依据 |
| 采购/供应链 | 比较不同供应地和工厂 | 不清楚从中国迁到越南/墨西哥是否真的省税 | 多原产地并排比较，展示成本差异和风险 |
| 财务/经营者 | 审批订单、复盘毛利 | 报价低估关税导致亏损 | 结果可追溯，能看到每层税种和假设 |
| 跨境电商运营 | 选品、备货、核算美国成本 | SKU 多，手工查税率慢 | Excel 批量测算和导出报告 |
| 贸易合规负责人 | 跟踪政策变化 | 关税政策频繁变动，影响范围难判断 | HS/路线 watchlist 和通知推送 |
| 开发者/企业系统 | 内部 ERP/报价系统集成 | 需要结构化税率数据和批量接口 | 后续 REST API、Webhook 和批量任务 |

## 三、产品原则

1. **可信优先于功能数量**：宁可少算，也不能把不确定项伪装成确定结果。
2. **解释优先于黑盒结果**：每个结果必须说明税种、依据、来源、有效期和缺失项。
3. **中国业务流优先**：优先适配中文、Excel、1688/报价表、采购决策、老板审批场景。
4. **批量优先于单次查询**：真正的商业价值来自 SKU 批量、路线批量和政策变更批量影响分析。
5. **政策监控优先于静态查询**：市场已经从“查一次税率”转向“持续监控政策风险”。
6. **AI 只能辅助，不能替代合规判断**：AI 需要给候选、依据、置信度和澄清问题。

## 四、核心用户场景

### 场景 1：单 SKU 多原产地测算

用户输入：

- HS/HTS 编码或商品描述
- 货值
- 原产地候选国家/地区
- 运输方式、运费、保险费
- 预计入境日期

系统输出：

- 每个原产地的 landed cost
- MFN/FTA、Section 301、Section 232、Section 122、AD/CVD 风险、MPF/HMF 等拆解
- 最低成本路线和节省金额
- 缺失项与人工确认提示
- 数据来源和更新时间

验收标准：

- 用户能在 1 分钟内完成一次多原产地比较。
- 结果页能解释每层税种为什么适用或不适用。
- 不确定 AD/CVD 不自动计入确定总成本。

### 场景 2：Excel 批量报价/采购表测算

用户上传包含 SKU、HS/HTS、货值、原产地、运费等字段的 Excel。

系统输出：

- 每行 SKU 的 landed cost
- 多原产地比较结果
- 成本最低路线
- 高风险 SKU 列表
- 缺失字段列表
- 可导出的中文报告

验收标准：

- 支持模板下载。
- 支持字段映射和错误行提示。
- 100 行以内文件可稳定完成测算。
- 导出报告可用于内部审批或客户沟通。

### 场景 3：政策变化订阅与影响分析

用户设置 watchlist：

- HS/HTS 编码
- 原产地
- 税种
- 成本变化阈值
- 通知渠道

系统在政策或数据变化后输出：

- 哪些 HS/路线受影响
- 成本变化金额和百分比
- 政策来源和生效时间
- 建议复核动作

验收标准：

- 支持邮件、钉钉、企业微信、Webhook。
- 同一变化不会重复轰炸。
- 通知必须包含来源、时间、影响范围和建议动作。

### 场景 4：历史 `as_of` 订单复盘

用户输入历史订单的入境日期、HS/HTS、原产地和货值。

系统输出：

- 当时适用的税率和政策
- 与当前税率的差异
- 可能涉及的退款、补税或复盘提示

验收标准：

- 税率计算支持入境日期。
- 历史结果明确标注数据版本和来源。
- 对缺失历史数据给出“不确定”而不是默认当前税率。

## 五、功能优先级

### P0-A：政策正确性热修

目标：避免当前产品输出明显过时或误导性的计算结果。

| 需求 | 描述 | 验收标准 |
|------|------|----------|
| IEEPA 历史化 | 当前计算不再默认计入 `reciprocalTariff`。 | 当前日期下 IEEPA 不进入默认总税额；历史查询可保留。 |
| Section 122 最小支持 | 新增 Section 122 税种，支持生效日期、期限、豁免、叠加/排除规则。 | 不同入境日期可切换 IEEPA 历史和 Section 122 当前状态。 |
| De minimis 全球暂停 | 默认美国 de minimis 全球暂停，保留 CN/HK 先行暂停历史节点。 | 低于 $800 的美国进口不再默认免税。 |
| AD/CVD 风险提示 | 不确定 AD/CVD 不自动计入总成本。 | 结果显示“可能适用，需人工确认”。 |
| 文案修正 | 修正“对等关税”“CN/HK 已撤销”“AI 推荐最优方案”等口径。 | UI、i18n、报告口径一致。 |

### P0-B：可信计算模型

目标：让每个税种都可解释、可追溯、可测试。

| 需求 | 描述 | 验收标准 |
|------|------|----------|
| `TariffMeasure` | 统一税种模型，包含 type、legalBasis、chapter99Code、rate、effectiveDate、expiryDate、stackingRule、exemptions、sourceUrl、confidence。 | 所有税种都走统一结构。 |
| 来源与有效期 | 每层税种显示来源、抓取时间、生效日期、失效日期。 | 结果页可追溯每个数字。 |
| 缺失项机制 | 材料占比、生产商、AD/CVD scope 等缺失时显式提示。 | 不用静默默认值伪造确定性。 |
| 回归测试样本 | 建立 10-20 个固定 HTS10 + 原产地 + 日期样本。 | 核心叠加规则有自动化测试。 |

### P1：官方数据管道与健康看板

目标：从演示数据升级为可运营数据服务。

数据源优先级：

1. USITC HTS / HTSUS
2. USTR Section 301
3. CBP 执行指引与 de minimis
4. Federal Register
5. ITA ADCVD

| 需求 | 描述 | 验收标准 |
|------|------|----------|
| HTS 基础数据导入 | 导入 HTS10、基础税率、special rate、Chapter 99 关联。 | 覆盖核心章节，不再依赖 5 个 mock 品类。 |
| 政策变化采集 | 采集 Section 301/232/122、AD/CVD、de minimis 相关变化。 | `tariff_change_log` 可展示政策时间线。 |
| 数据健康看板 | 展示每个数据源最后同步时间、失败次数、记录数、异常项。 | 用户和运营能判断数据是否过期。 |
| 数据版本 | 每次计算绑定数据版本或抓取时间。 | 历史结果可复盘。 |

### P2：批量工作流与中文报告

目标：形成面向中国企业的核心差异化。

| 需求 | 描述 | 验收标准 |
|------|------|----------|
| Excel 模板 | 提供标准导入模板。 | 用户可按模板批量上传 SKU。 |
| 字段映射 | 支持用户将自有列名映射到系统字段。 | 非标准表头也能导入。 |
| 批量测算 | 支持多 SKU、多原产地、多入境日期测算。 | 100 行以内稳定运行。 |
| 风险分层 | 标注高税率、高不确定性、缺失项、政策变化影响。 | 报告可直接用于内部复核。 |
| 中文导出报告 | 导出 Excel/PDF，包含假设、来源、税种拆解、风险和建议。 | 报告能给老板、客户、采购、财务阅读。 |

### P3：订阅监控与试点商业化

目标：从一次性查询转向持续监控。

| 需求 | 描述 | 验收标准 |
|------|------|----------|
| Watchlist | 用户可按 HS/HTS、原产地、税种、阈值订阅。 | 可创建、编辑、暂停订阅规则。 |
| 通知渠道 | 支持邮件、钉钉、企业微信、Webhook。 | 通知包含来源、影响范围、变化金额、建议动作。 |
| 去重与频控 | 同一政策变化不重复轰炸。 | 同一事件按用户规则去重。 |
| 付费灰度 | Starter/Pro 用户开放批量和订阅能力。 | 套餐限制与核心价值挂钩，不以签到为主卖点。 |

### P4：AI 分类与 API 扩展

进入条件：P0/P1 数据可信度、来源追溯和回归样本稳定。

| 需求 | 描述 | 验收标准 |
|------|------|----------|
| AI HS/HTS 候选 | 根据商品描述、材质、用途、图片/发票字段返回候选 HTS10。 | 返回 3 个候选、置信度、依据和澄清问题。 |
| API | 提供 `/resolve`、`/compare`、`/batch`、`/watch`、`/changes`。 | 返回每层税种、来源、有效期、缺失项和警告。 |
| Webhook | 企业用户接收政策变化事件。 | 与 watchlist 和通知服务打通。 |
| 电商插件评估 | 只做 duty preview，不承诺 DDP/代缴。 | 不偏离核心测算与监控主线。 |

## 七、结果页核心字段

单次测算结果至少应包含：

- HS/HTS 编码
- 商品描述
- 原产地
- 入境日期
- 货值、运费、保险费
- MFN / FTA 税率与金额
- Section 301 税率与金额
- Section 232 税率与金额或不适用原因
- Section 122 税率与金额或不适用原因
- AD/CVD 风险提示
- MPF / HMF
- 总税费
- landed cost
- 有效税率
- 缺失项
- 数据来源
- 数据更新时间
- 置信度
- 免责声明

## 八、商业化方向

### 1. 免费版

适合试用和低频用户。

- 少量单次测算
- 基础税种拆解
- 限制批量导出
- 限制历史查询
- 不开放订阅和 webhook

### 2. Starter

适合个人外贸、采购和小团队。

- 更多单次测算
- 本地多原产地对比
- 少量 Excel 导出
- 少量 watchlist
- 中文报告基础版

### 3. Pro

适合中小贸易公司和跨境电商团队。

- 批量 Excel 测算
- 更多 watchlist
- 政策变化通知
- 历史 `as_of` 查询
- 中文报告高级版
- 团队内共享

### 4. Enterprise

适合有内部系统和合规流程的企业。

- API
- Webhook
- 私有数据源配置
- 团队席位
- 审计日志
- SLA 和专属支持

## 十、风险与边界

| 风险 | 应对 |
|------|------|
| 政策变化过快 | 显示数据更新时间、适用日期和数据健康状态。 |
| AD/CVD 误算 | P0 只提示风险，不自动计入确定总成本。 |
| AI 分类误导 | 只返回候选和依据，不做最终归类结论。 |
| 用户误认为法律意见 | 所有报告和结果页显示免责声明。 |
| 数据源不可用 | 保留同步状态、失败记录和降级提示。 |
| 竞品快速跟进中文市场 | 强化 Excel、1688/报价表、中国采购流和中文报告。 |

## 十二、近期研发切入顺序

1. 修正 `reciprocalTariff` 当前默认逻辑。
2. 新增 Section 122 最小税种模型。
3. 修正 de minimis 默认规则。
4. 把 AD/CVD 改为风险提示，不作为确定税额。
5. 统一文案：IEEPA 历史、Section 122 临时附加税、de minimis 全球暂停。
6. 设计 `TariffMeasure` 并迁移现有税种。
7. 增加来源、有效期、置信度、缺失项字段。
8. 建立核心回归测试样本。
9. 做 USITC HTS 数据导入。
10. 做 Excel 批量测算和中文报告。
