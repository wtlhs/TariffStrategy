## 一、架构总览

```
┌─────────────────────────────────────────────────────────┐
│                    Chrome Extension (MV3)                │
├──────────────┬──────────────────┬───────────────────────┤
│  popup.html  │  options.html    │  background (SW)      │
│  策略对比页   │  数据管理+订阅    │  定时任务+通知推送      │

├──────────────┴──────────────────┴───────────────────────┤
│                   共享层 (shared/)                        │
│  components / hooks / services / store / i18n / types   │
├─────────────────────────────────────────────────────────┤
│                   持久化层                                │
│          chrome.storage.local / chrome.alarms            │
├─────────────────────────────────────────────────────────┤
│                   外部服务                                │
│     AI 税率服务 (FastAPI) / 爬虫数据源 / 通知渠道          │
└─────────────────────────────────────────────────────────┘
```

### 核心架构决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| Manifest 版本 | **MV3** | MV2 已于 2025.06 全面停用 |
| 构建工具 | **Vite + @crxjs/vite-plugin** | 零配置多入口、HMR 支持、轻量 |
| 前端框架 | **React 18 + TypeScript** | 与源项目一致，组件可直接移植 |
| 样式方案 | **TailwindCSS 3.4+** | 与源项目样式类名完全一致 |
| 状态管理 | **Zustand + chrome.storage.local** | 轻量、天然支持持久化中间件 |
| 图标库 | **lucide-react** | 与源项目一致 |
| 后端通信 | **直连 AI 服务** | 去掉 bi-backend 代理层 |
| 认证方式 | **API Key (chrome.storage)** | 无需 chrome.identity，简单直接 |

## 四、核心模块设计

### 4.1 策略计算引擎（纯前端，离线可用）

```typescript
// lib/strategy-engine.ts
interface StrategyInput {
  hsCode: string
  origin: string
  destination: string
  goodsValue: number
  routes: RouteData[]
}

interface RouteData {
  routing: string       // "RO→US"
  taxRate: number       // 0.035
  shippingDays: number
  shippingCost: number
}

function calculateStrategies(input: StrategyInput): StrategyResponse {
  const results = input.routes.map(route => ({
    routing: route.routing,
    taxCost: input.goodsValue * route.taxRate,
    shippingCost: route.shippingCost,
    taxRate: route.taxRate,
    totalCost: input.goodsValue * (1 + route.taxRate) + route.shippingCost,
  }))

  results.sort((a, b) => a.totalCost - b.totalCost)

  const bestTotal = results[0].totalCost
  return {
    success: true,
    best_strategy: { ...results[0], is_best: true, savings_vs_best: 0 },
    all_strategies: results.map((r, i) => ({
      ...r,
      is_best: i === 0,
      savings_vs_best: i === 0 ? 0 : r.totalCost - bestTotal,
    })),
    ai_suggestion: generateSuggestion(results),
  }
}
```

**关键点：** 策略计算完全在本地完成（使用模拟数据），无需网络请求。当 AI 服务可用时，可切换为远程模式获取真实税率和 AI 建议。

### 4.2 状态持久化

```typescript
// store/strategy-store.ts
import { create } from 'zustand'
import { persist, createJSONStorage } from 'zustand/middleware'

// 自定义 chrome.storage 适配器
const chromeStorageAdapter = {
  getItem: (key: string) =>
    chrome.storage.local.get(key).then(r => r[key] ?? null),
  setItem: (key: string, value: string) =>
    chrome.storage.local.set({ [key]: value }),
  removeItem: (key: string) =>
    chrome.storage.local.remove(key),
}

const useStrategyStore = create(
  persist(
    (set) => ({
      searchParams: null,
      strategyResult: null,
      setSearchParams: (params) => set({ searchParams: params }),
      setStrategyResult: (result) => set({ strategyResult: result }),
      reset: () => set({ searchParams: null, strategyResult: null }),
    }),
    {
      name: 'tariff-strategy-state',
      storage: createJSONStorage(() => chromeStorageAdapter),
    }
  )
)
```

### 4.4 API 客户端（可切换本地/远程模式）

```typescript
// services/api-client.ts
type Mode = 'local' | 'remote'

const getBaseUrl = async (): Promise<string> => {
  const config = await chrome.storage.local.get('apiConfig')
  return config.apiConfig?.baseUrl ?? 'http://localhost:8000'
}

const getApiKey = async (): Promise<string> => {
  const config = await chrome.storage.local.get('apiConfig')
  return config.apiConfig?.apiKey ?? ''
}

async function request<T>(path: string, options?: RequestInit): Promise<T> {
  const baseUrl = await getBaseUrl()
  const apiKey = await getApiKey()

  const res = await fetch(`${baseUrl}${path}`, {
    ...options,
    headers: {
      'Content-Type': 'application/json',
      'X-API-Key': apiKey,
      ...options?.headers,
    },
  })

  if (!res.ok) throw new Error(`API Error: ${res.status}`)
  const data = await res.json()
  if (!data.success) throw new Error(data.message ?? 'Unknown error')
  return data.data
}
```

## 六、与源项目的差异适配

| 原始设计 | Chrome 插件适配 | 变更原因 |
|---------|----------------|---------|
| Next.js App Router | Vite SPA（popup + options） | Chrome 扩展不支持 SSR |
| sessionStorage | chrome.storage.local | 跨页面持久化 |
| bi-backend 代理 | 直连 AI 服务 | 减少中间层 |
| Bearer Token 认证 | API Key + chrome.storage | 简化认证 |
| popup 800px 限制 | 精简布局 + "查看更多"链接 | popup 尺寸限制 |
| Excel 导入导出 | File API + SheetJS | 无后端依赖 |
| 页面路由 `/tariff-*` | popup / options 切换 | 扩展页面模型不同 |
| 全屏宽度布局 | popup: 800px / options: 全屏 | popup 限制 |

## 八、技术风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| popup 800px 宽度限制 | UI 布局需精简 | 主数据放 options，popup 只展示核心结果 |
| @crxjs/vite-plugin 维护风险 | 构建可能出问题 | 备选方案：WXT 框架或手动 Vite rollup 配置 |
| AI 服务不可用 | 无真实税率数据 | 默认使用本地模拟数据，提供降级模式 |
| Service Worker 5分钟超时 | 长时间任务中断 | 使用 chrome.alarms 而非 setTimeout |
| MV3 CSP 限制 | 无内联脚本/样式 | Tailwind 编译输出安全，避免运行时样式生成 |
