### 1.2 本地策略对比的积分逻辑矛盾

**问题：** `商业化文档` 对"本地策略对比"的积分消耗定义自相矛盾。

| 位置 | 描述 |
|------|------|
| 积分消耗表 | "策略对比（本地模拟）— 2 积分" |
| 免费版额度 | "每日 3 次" |
| 基础版月度估算 | "本地对比无限 × 2分 = **0**（已含在套餐内）" |
| 套餐对比总表 | 基础版"本地策略对比 = **无限**" |

**核心矛盾：** 付费用户的本地对比到底消不消耗积分？
- 如果"无限"= 不消耗积分，那消耗表里就不该写"2 积分"
- 如果"无限"= 无限次数但每次仍扣 2 积分，那月度估算应该是"每月 800 积分 ÷ 2 = 400 次"而不是"0"

**建议：** 明确规则 ——
```
免费用户：每日 3 次（每次消耗 2 积分）
付费用户：本地对比不消耗积分，无限使用（只有 AI 分析才消耗积分）
```

### 1.3 Popup 空间预算不足

**问题：** UI 文档设定 popup 为 460×580px，内容区 480px。但 Tab1"策略对比"的元素高度超标。

| 元素 | 预估高度 |
|------|---------|
| 检索条件区（4个输入 + 按钮） | ~220px |
| 最优结果摘要行 | ~36px |
| 7行对比表（每行 ~28px） | ~196px |
| AI 建议 1 行 | ~30px |
| 底部 3 个按钮 | ~40px |
| 卡片内边距 + 间距 | ~40px |
| **合计** | **~562px** |

超出内容区 480px，用户必须滚动才能看到完整结果和底部按钮。

**建议：**
- **方案 A：** 对比表默认只显示前 3 名 + "展开全部 7 条"按钮（折叠 4 行节省 ~112px）
- **方案 B：** popup 高度调到 600px（Chrome 最大值），内容区 500px，仍需少量滚动
- **推荐方案 A**，让首屏完整展示核心信息

## 二、中等问题（建议修复，不阻塞但影响质量）

### 2.1 技术选型需重新评估

**问题：** 架构文档推荐 `@crxjs/vite-plugin`，但前期调研已发现该插件在寻求新维护者，存在长期风险。

| 方案 | 优势 | 风险 |
|------|------|------|
| @crxjs/vite-plugin | 轻量、HMR | 维护者出走，兼容性隐患 |
| WXT | 跨浏览器、活跃维护 | 学习成本、框架约束 |
| Vite 手动 rollup 配置 | 零依赖、完全控制 | 配置繁琐 |

**建议：** 优先考虑 **WXT**（all-api-hub 也使用了 WXT），与项目的技术栈一致（React + TypeScript + TailwindCSS）。如果坚持用 @crxjs，需要准备回退方案。

---

### 2.2 全屏页 15 个子页面过多

**问题：** UI 文档的 options 全屏页设计了 15 个侧边栏页面。对于 Chrome 扩展的 options 页，这个数量偏多。

- 用户初次打开 options 看到如此长的侧边栏会产生选择焦虑
- 部分页面功能重叠（如"导入导出"和"产品管理"可合并）
- 开发工作量被低估

**建议：** 合并为 8 个核心页面：

```
原 15 个页面              →  合并为 8 个
─────────────────────────────────────
策略对比 + 趋势 + 历史     →  策略分析（含趋势图和历史）
产品管理 + 发货国家         →  基础配置（产品和发货国家）
采集数据 + 导入导出         →  数据中心（采集数据 + Excel操作）
订阅规则 + 通知渠道         →  订阅管理
个人信息                   →  账户设置（合并个人信息+系统设置）
套餐管理 + 积分中心         →  套餐与积分
系统设置                   →  （合并到账户设置）
```

---

### 2.3 收入预估偏乐观

**问题：** 商业文档预估 5% 付费转化率，但行业 SaaS freemium 平均仅 2-4%。

| 指标 | 文档预估值 | 行业参考值 | 差异 |
|------|----------|----------|------|
| 付费转化率 | 5% | 2-4% | 偏高 25-150% |
| 月收入（6个月） | ¥6,105 | ¥2,440-4,880 | 可能偏高 |

**建议：** 提供悲观/中性/乐观三档预估；同时补充"盈亏平衡点"分析（服务器 + AI API 成本 vs 收入）。

---

### 2.4 免费版限制可能过早劝退用户

**问题：** 免费用户"每日 3 次本地对比"可能在试用阶段就触达上限。如果用户第一天认真测试（对比不同 HS编码、不同货值），3 次很容易用完。

**建议：** 免费版改为 **每周 15 次**（而非每日 3 次），给用户更灵活的使用节奏。或者首周不限次数，第二周开始限制。

---

### 2.5 签到连续天数归零规则未定义

**问题：** 商业文档定义了连续签到 3/7/30 天的加成，但未定义"断签"规则。

- 如果一天没签，是否归零？
- 是否支持"补签"（消耗积分补签）？
- 周末/节假日是否跳过？

**建议：**
```
断签规则：连续天数归零，重新从第 1 天开始
补签功能：消耗 20 积分可补签 1 天（每月最多 3 次）
```

## 三、轻微问题（可后续优化）

### 3.1 缺少错误状态设计

所有文档都只描述了"正常流程"，没有设计错误场景的 UI：

| 错误场景 | 需要的设计 |
|---------|----------|
| AI 服务不可用 | 本地模式降级提示 |
| 网络断开 | 离线状态指示 |
| API Key 无效 | 配置引导弹窗 |
| 数据加载中 | 骨架屏 / Loading 动画 |
| 爬虫数据为空 | 空状态插图 |
| 税率数据过期 | 数据时效提示 |

### 3.2 缺少 Chrome Web Store 上架准备

| 缺失项 | 说明 |
|--------|------|
| 商店标题/描述 | 中英文各需准备 |
| 截图素材 | 1280×800 或 640×400，至少 1 张 |
| 宣传图片 | 440×280 小宣传图 |
| 隐私政策 URL | Chrome Web Store 强制要求 |
| 单次使用权限说明 | host_permissions 需要解释 |

### 3.3 Welcome Page Step 3 的交互矛盾

UI 文档 Step 3 中有"开始对比"按钮，但标注"结果会在弹窗中展示"。问题：Welcome Page 是一个新 Tab 页，不是 popup。用户点击按钮后需要：
1. 先引导用户关闭 Tab
2. 点击工具栏图标打开 popup
3. 结果在 popup 中展示

这个流程过于复杂。

**建议：** Step 3 改为在 Welcome Tab 页内直接展示模拟结果（内嵌 Demo），而非跳转到 popup。

### 3.4 options 页面缺少响应式设计

Options 全屏页采用左侧边栏 200px + 右侧自适应的布局，但没有考虑窄窗口场景（如用户把浏览器窗口缩小）。建议侧边栏在窄屏时折叠为图标模式。

### 3.5 数据模型字段命名风格不统一

| 文档 | 风格 | 示例 |
|------|------|------|
| 原始规格 | snake_case | `hs_code`, `total_cost`, `tax_rate` |
| 架构文档 | camelCase | `hsCode`, `totalCost`, `taxRate` |
| 商业文档 | camelCase | `userId`, `planExpiresAt` |

**建议：** 统一为 camelCase（TypeScript 惯例），API 层做 snake_case → camelCase 转换。

## 四、遗漏项

| 遗漏 | 重要程度 | 说明 |
|------|---------|------|
| 离线模式 | 高 | 核心功能（本地对比）应完全离线可用，需明确离线范围 |
| 数据备份恢复 | 中 | 用户换设备/重装后如何恢复产品和配置数据 |
| 无障碍 (A11y) | 中 | 键盘导航、屏幕阅读器支持、对比度检查 |
| 国际化 (i18n) 执行细节 | 中 | 原始规格有 70 个 key，商业化和 UI 新增了积分/套餐/签到相关 key，未更新 key 列表 |
| CI/CD 流水线 | 低 | 自动构建、自动测试、自动发布 Chrome Web Store |
| 版本更新策略 | 低 | 扩展更新时如何通知用户、如何迁移数据 |
| 用户反馈渠道 | 低 | Bug 报告入口、功能建议渠道 |
| 隐私政策 & 数据安全 | 高 | Chrome Web Store 上架强制要求；积分/套餐涉及用户数据 |
| 邀请好友机制 | 中 | 商业文档提到"邀请好友 +50 积分"，但未设计邀请链接/分享机制 |
| 税率数据更新频率 | 低 | 爬虫多久运行一次？数据有效期多久？ |
