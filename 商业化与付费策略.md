## 一、商业化核心思路

```
┌─────────────────────────────────────────────────────────┐
│                    积分经济模型                           │
│                                                         │
│   签到 / 任务 ──→ 赚积分 ──→ 消耗积分 ──→ 查询/AI分析    │
│        ↑                              ↓                 │
│   付费购买积分 ◄─────────── 积分不足时引导 ◄──            │
│                                                         │
│   订阅制 = 每月固定积分 + 专属功能 + 更高速率              │
└─────────────────────────────────────────────────────────┘
```

**核心理念：** 免费 + 积分 + 订阅 三层模型

- **免费层**：签到赚积分，满足轻度用户日常使用
- **积分制**：所有消耗性操作按次计费（查询、AI 分析、数据导出）
- **订阅制**：包月/包年，固定积分 + 专属功能 + 优先级

**为什么选积分制而非纯订阅？**
1. 国际贸易用户的查询频率差异极大（有人每天查，有人一周查一次）
2. 积分制让轻度用户不被"月费"劝退，重度用户直接买包年
3. 签到机制提升日活（参考 all-api-hub 自动签到）
4. 积分余额可见，用户感知"花多少得多少"，付费转化率更高

## 七、转化漏斗设计

### 7.1 用户旅程

```
安装插件 → 首次注册(+100积分) → 完成引导任务(+50积分)
     ↓
体验核心功能(策略对比) → 消耗积分 → 积分不足提示
     ↓
引导签到(每日10积分) → 连续签到加成 → 形成习惯
     ↓
触及免费额度限制 → 弹窗推荐套餐 → 7天试用专业版
     ↓
试用到期 → 降级为免费 → 高级功能灰化(可见不可用)
     ↓
付费转化 → 按月续费 → 年付优惠引导
```

### 7.2 关键转化触点

| 触点 | 时机 | 行为 |
|------|------|------|
| 积分不足弹窗 | 积分 < 操作消耗 | 显示"签到赚积分"或"升级套餐" |
| 免费额度用完 | 达到每日/每月上限 | "今日免费额度已用完，升级解锁无限" |
| 功能灰化点击 | 点击付费功能 | 弹出套餐对比表 + 7天试用入口 |
| 签到页面底部 | 每日签到后 | "专业版签到 2x 加成，每月多得 500 积分" |
| 试用期到期前 3 天 | 试用倒计时 | 推送通知"您的专业版试用还剩 3 天" |
| 首次 AI 分析后 | 看到分析结果 | "升级专业版，获取深度趋势分析" |
| 月底续费提醒 | 订阅到期前 5 天 | 邮件/系统通知续费优惠 |

### 7.3 7天免费试用策略

- 注册即送专业版 7 天试用（需绑定邮箱）
- 试用期内享受专业版全部功能
- 试用到期自动降级为免费版，**不自动扣费**
- 试用期内消耗的积分不退还，但额外赠送的积分保留
- 试用结束后展示用量报告："这 7 天您节省了 ¥X,XXX 运费"

### 8.2 积分消耗中间件

```typescript
// services/credit-guard.ts
async function consumeCredits(
  userId: string,
  amount: number,
  reason: string
): Promise<{ success: boolean; balance: number }> {
  const account = await getAccount(userId)

  // 免费用户检查每日额度
  if (account.plan === 'free') {
    const dailyUsage = await getDailyUsage(userId, reason)
    if (dailyUsage >= getFreeDailyLimit(reason)) {
      return { success: false, balance: account.credits }
    }
  }

  // 检查积分余额
  if (account.credits < amount) {
    return { success: false, balance: account.credits }
  }

  // 扣减积分
  const newBalance = account.credits - amount
  await updateBalance(userId, newBalance)
  await createTransaction(userId, 'spend', amount, reason, newBalance)

  return { success: true, balance: newBalance }
}
```

### 8.3 签到服务

```typescript
// services/check-in.ts
async function dailyCheckIn(userId: string): Promise<CheckInResult> {
  const account = await getAccount(userId)
  const today = getDateString()

  // 检查是否已签到
  const existing = await getCheckInRecord(userId, today)
  if (existing) throw new Error('ALREADY_CHECKED_IN')

  // 计算连续天数
  const yesterday = getDateString(-1)
  const yesterdayRecord = await getCheckInRecord(userId, yesterday)
  const newStreak = yesterdayRecord ? account.checkInStreak + 1 : 1

  // 计算积分
  const baseCredits = 10
  const streakBonus = calculateStreakBonus(newStreak) // 0/5/10/20
  const planMultiplier = getPlanMultiplier(account.plan) // 1/1.5/2/3
  const totalCredits = Math.round((baseCredits + streakBonus) * planMultiplier)

  // 发放积分
  await updateBalance(userId, account.credits + totalCredits)
  await updateStreak(userId, newStreak)
  await createCheckInRecord(userId, today, baseCredits, streakBonus, planMultiplier - 1, totalCredits)
  await createTransaction(userId, 'earn', totalCredits, 'check_in', account.credits + totalCredits)

  return {
    creditsEarned: totalCredits,
    newStreak,
    streakBonus,
    newBalance: account.credits + totalCredits,
  }
}

function calculateStreakBonus(streak: number): number {
  if (streak >= 30) return 20
  if (streak >= 7) return 10
  if (streak >= 3) return 5
  return 0
}

function getPlanMultiplier(plan: string): number {
  switch (plan) {
    case 'enterprise': return 3
    case 'pro': return 2
    case 'starter': return 1.5
    default: return 1
  }
}
```
